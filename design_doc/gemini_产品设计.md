这是一个基于**产品设计师 (Product Designer)** 视角为您定制的提示词（Prompts）套件。

这套提示词旨在帮助您利用 AI（如 ChatGPT、Claude）生成**产品需求文档 (PRD)、线框图描述、用户体验流程 (User Journey)** 以及针对 Electron + Python 架构的**界面交互规范**。

您可以按顺序使用，也可以根据当前开发阶段挑选使用。

---

### 第一阶段：产品定义与用户画像 (Product Definition)

**目标**：明确“谁在用”以及“核心痛点”，确保技术选型（本地 RAG + Python 生成）真正服务于业务。

> **Prompt 1：用户画像与核心痛点**
> “我正在设计一款基于 Electron 和本地 AI（RAG）的标书生成工具。目标用户是‘售前工程师’和‘投标专员’。
> 请作为一名资深产品经理，为我生成一份详细的**用户画像 (User Persona)**。
> 重点描述他们在处理招标文件（RFP）时的痛点，例如：
> 1. 手动从几百页 PDF 中寻找技术参数的痛苦。
> 2. 复制粘贴历史标书导致格式错乱的问题。
> 3. 对数据隐私（本地部署）的强诉求。
> 并基于这些痛点，列出本工具必须具备的 5 个‘杀手级功能’（MVP 功能列表）。”
> 
> 

> **Prompt 2：产品价值主张 (Value Proposition)**
> “基于 Electron + Vue3 (前端) 和 Python (后端) 的技术架构，我们的核心优势是‘极致的文档处理能力’（利用 Python 生态）和‘数据隐私安全’。
> 请为这款产品写一段简短有力的**产品价值主张**（Elevator Pitch），以及一段用于软件启动页（Splash Screen）的欢迎文案，强调‘本地智能’和‘专业文档生成’的结合。”

---

### 第二阶段：UX/UI 布局与交互框架 (Layout & Interaction)

**目标**：设计符合 Electron 桌面应用习惯的界面，特别是处理双进程（Node.js + Python Sidecar）带来的状态反馈。

> **Prompt 3：桌面应用布局设计 (Layout Strategy)**
> “请作为一名 UI/UX 设计师，为这款标书生成工具构思一个**三栏式桌面应用布局**。
> 1. **左侧栏 (Sidebar)**：用于项目管理、知识库（向量库）管理、模板库。
> 2. **中间栏 (Chat/Work Area)**：核心的 RAG 交互区，用户与 AI 对话，检索历史标书条款。
> 3. **右侧栏 (Context/Preview)**：实时预览 PDF 源文件（高亮引用位置）或生成的 Word 文档预览。
> 请用文字详细描述每个区域的功能组件、布局比例，以及如何利用 Vue3 的组件化思想来组织这些区域。”
> 
> 

> **Prompt 4：Python 后端状态的交互反馈**
> “由于本软件依赖一个本地启动的 Python 后端（用于 RAG 和生成），启动和运行可能需要时间，或者会占用 CPU。
> 请设计一套**系统状态反馈机制**（System Feedback UI）：
> 1. 当应用启动，Python 服务正在拉起时，界面应如何显示？（避免白屏）
> 2. 当正在进行耗时的 RAG 检索或 Word 生成时，如何优雅地展示进度？
> 3. 如果 Python 进程崩溃，前端应如何提示用户并引导重启服务？
> 请提供具体的 UI 交互文案和视觉提示建议。”
> 
>

### 第三阶段：核心功能体验设计 (Core Features UX)

**目标**：针对 RAG 和 文档生成这两个最难的交互点进行深度设计。

> **Prompt 5：RAG 检索结果的可解释性设计**
> “用户非常介意 AI 编造数据（幻觉）。请设计一个**‘可信度校验’交互组件**：
> 当用户问‘我们的服务器最大支持多少并发？’AI 给出答案后，界面上应如何展示：
> 1. 答案的引用来源（具体是哪份历史标书的第几页）。
> 2. 点击引用来源后，右侧预览窗口应如何联动（例如：自动滚动到 PDF 对应位置并高亮段落）。
> 请描述这个交互流程（User Flow）和界面元素。”
> 
> 

> **Prompt 6：Word 模板映射的交互界面**
> “我们的后端使用 `docxtpl` (Python) 填充 Word 模板。前端需要一个界面来让用户‘配置模板’。
> 请设计一个**‘模板变量映射器’ (Template Mapper)** 的 UI 方案：
> 1. 用户上传一个带有 Jinja2 标签（如 `{{ project_name }}`）的 Word 模板。
> 2. 系统自动解析出所有变量列表。
> 3. 用户可以在界面上为每个变量指定数据来源（是手动输入、还是来自 RAG 检索结果、还是固定系统参数）。
> 请详细描述这个配置界面的布局和操作逻辑。”
> 
> 

---

### 第四阶段：视觉风格与设计系统 (Visual Style)

**目标**：确定产品的视觉基调。

> **Prompt 7：B 端工具的视觉风格定义**
> “这款工具是给专业人士使用的生产力工具。请定义一套**设计系统 (Design System)** 的基础规范：
> 1. **色彩体系**：推荐一套专业、冷静且护眼的配色方案（支持深色模式 Dark Mode）。
> 2. **字体排版**：考虑到大量文本阅读，请给出字体大小、行高和阅读区域宽度的建议。
> 3. **组件风格**：推荐使用哪种现成的 Vue3 UI 库（如 Element Plus, Naive UI, Antdv）最适合此类复杂表单和数据展示工具？并说明理由。”
> 
> 

---

### 第五阶段：特殊场景与边缘情况 (Edge Cases)

**目标**：覆盖桌面软件特有的问题。

> **Prompt 8：离线与资源受限模式**
> “考虑到用户可能在无网环境下办公，或者电脑配置较低。
> 请设计以下场景的**降级体验 (Graceful Degradation)**：
> 1. **无网模式**：当无法连接 OpenAI API 时，如何引导用户切换到本地模型（如 Ollama）或仅使用基于关键词的传统搜索？
> 2. **资源占用**：当本地向量数据库（LanceDB）正在写入大量数据导致 CPU 飙升时，前端如何安抚用户并保持 UI 响应不卡顿？”
> 
>

### 建议的使用方式：

1. **先用 Prompt 1 和 2** 来“洗脑” AI，让它完全理解你的产品背景。
2. **重点攻克 Prompt 6**，因为这是你技术选型（Python docxtpl）在前端最难表现的部分。你需要一个可视化的界面来管理那些复杂的 Jinja2 标签，否则用户体验会很差。
3. **用 Prompt 3 生成 Mermaid 流程图**，直接贴到你的开发文档中。
