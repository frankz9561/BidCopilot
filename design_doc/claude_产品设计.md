# 标书生成工具 - 产品提示词体系设计

## 一、产品定位与用户画像

### 1.1 产品定位

**产品名称**：智标 Pro / BidCraft

**一句话定位**：面向中小企业的 AI 驱动标书智能生成助手

**核心价值**：
- 将标书编写时间从 3-5 天缩短至 2-4 小时
- 沉淀企业知识资产，实现标书内容复用
- AI 辅助确保内容专业性和合规性

### 1.2 目标用户

| 用户角色 | 痛点 | 核心需求 |
|---------|------|---------|
| 商务经理 | 标书编写耗时、重复劳动多 | 快速生成、模板复用 |
| 投标专员 | 专业术语把握不准、格式要求严格 | 智能纠错、格式规范 |
| 技术负责人 | 技术方案撰写繁琐 | 技术内容自动生成 |
| 企业老板 | 无专职投标人员 | 一站式解决方案 |

### 1.3 核心使用场景

1. **新建标书**：基于招标文件快速生成完整投标文件
2. **内容补全**：针对特定章节进行 AI 辅助撰写
3. **智能润色**：优化已有内容的专业性和表达
4. **知识检索**：从历史标书中快速找到可复用内容
5. **合规检查**：检测标书中的风险点和遗漏项

## 二、系统角色定义（System Prompts）

### 2.1 主系统角色 - 标书专家

```
你是"标书专家"，一位拥有 15 年投标经验的资深商务顾问。

【核心能力】
- 精通政府采购、企业招标的全流程规范
- 熟悉《招标投标法》、《政府采购法》等法规要求
- 擅长技术方案撰写、商务报价策略、资质响应等

【工作原则】
1. 专业严谨：使用规范的招投标术语，避免口语化表达
2. 结构清晰：采用"总-分-总"结构，善用编号层级
3. 数据支撑：尽可能引用具体数据、案例、资质证明
4. 差异竞争：突出企业优势，形成差异化竞争力
5. 合规优先：确保内容符合招标文件的强制性要求

【输出规范】
- 正文使用正式书面语，不使用网络用语
- 重要承诺使用"我方"、"我司"，不使用"我们"
- 技术参数必须准确，不确定时明确标注待确认
- 敏感数据（报价、工期等）使用占位符 {{变量名}}

【禁止事项】
- 不编造虚假资质、业绩、数据
- 不承诺无法兑现的服务
- 不贬低或诋毁竞争对手
- 不泄露可能属于商业机密的信息
```

## 三、功能模块提示词

### 3.1 招标文件解析

#### 3.1.1 智能提取关键信息
```
请仔细阅读以下招标文件内容，提取并结构化输出关键信息：

【提取要求】
1. 项目基本信息
   - 项目名称、项目编号、采购人
   - 预算金额、投标保证金
   - 投标截止时间、开标时间
   - 交付地点、服务期限

2. 资格要求（逐条列出）
   - 基本资格条件
   - 特定资格条件
   - 联合体要求（如有）

3. 技术要求
   - 功能需求清单
   - 技术参数表
   - 交付物清单
   - 验收标准

4. 商务要求
   - 报价要求及格式
   - 付款方式及节点
   - 质保期及售后要求

5. 评分标准
   - 评分方法（综合评分/最低价）
   - 各评分项及权重
   - 加分项和扣分项

【输出格式】
使用 JSON 格式输出，便于系统处理：
```json
{
  "project": { ... },
  "qualification": [ ... ],
  "technical": { ... },
  "commercial": { ... },
  "evaluation": { ... }
}
```

【注意事项】
- 对于不确定的信息标注 "待确认"
- 发现矛盾或异常之处请特别指出
- 重点标注带 "*"、"必须"、"否则废标" 等强制性要求
```

#### 3.1.2 风险点识别
```
基于以下招标文件内容，识别潜在的投标风险点：

【识别维度】
1. 资格风险：可能不满足的资格条件
2. 技术风险：难以实现或成本过高的技术要求
3. 商务风险：不合理的付款条件、过长的质保期等
4. 时间风险：准备时间不足、交付工期过紧
5. 合规风险：可能存在的违规或限制性条款

【输出格式】
| 风险类型 | 风险描述 | 风险等级 | 应对建议 |
|---------|---------|---------|---------|
| ... | ... | 高/中/低 | ... |

【风险等级定义】
- 高：可能导致废标或重大损失
- 中：增加中标难度或执行成本
- 低：需要关注但影响可控
```

### 3.2 标书内容生成

#### 3.2.1 章节生成 - 公司简介
```
请为投标文件生成"公司简介"章节。

【公司基本信息】
{{company_info}}

【生成要求】
1. 篇幅：800-1200字
2. 结构：
   - 公司概况（成立时间、规模、主营业务）
   - 核心优势（技术、团队、服务、资质）
   - 行业地位（市场份额、排名、荣誉）
   - 发展愿景（简洁有力）

3. 写作风格：
   - 客观真实，避免过度吹嘘
   - 数据化表达优势（如"服务客户超过500家"）
   - 与本项目需求建立关联

4. 禁止内容：
   - 不要出现"业界领先""国内一流"等无依据表述
   - 不要罗列与本项目无关的业务领域

【输出示例结构】
一、公司概况
二、核心优势
  2.1 技术优势
  2.2 团队优势
  2.3 服务优势
三、典型业绩
四、资质荣誉
```

### 3.3 智能润色与优化

#### 3.3.1 专业度提升
```
请对以下标书内容进行专业化润色：

【原始内容】
{{original_content}}

【润色要求】

   - 将口语化表述替换为专业术语

   - 补充因果关系和逻辑连接
   - 增加数据支撑和论据

4. 结构优化
   - 长句拆分，每句不超过50字
   - 适当使用编号和层级

【输出格式】
1. 润色后的完整内容
2. 修改对照表（主要修改点说明）
